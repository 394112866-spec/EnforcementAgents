# 案件分析 Skill（执行案件分析底稿）

## 功能概述

执行案件的分析不是从零开始的——你需要从一堆判决书、裁定书、申请书里提取关键信息，拼出完整的事实图景，找出对方藏在哪里、钱在哪里，以及你手上还缺什么。

本 skill 把"读材料→写底稿"这个过程结构化：通读 `file-sorter` 整理后的所有材料，提取标准化信息，建立精确到日的时间线，识别财产线索和隐匿手段，列出材料缺口，输出一份完整的案件分析底稿。

**产出可以直接作为内部办案笔记、给客户的汇报基础、以及后续 skill 的数据输入。**

## 使用场景

- 新委托执行案件，拿到材料后需要全面了解案情
- 终本案件恢复执行，需要重新梳理线索和缺口
- 执行中案件转阶段（如从查控转入评估拍卖），需要更新分析
- 向上级/客户汇报案件进展，需要一份结构化的案件底稿

## 输入

用户指定一个已整理好的案件材料文件夹路径。

**兼容两种输入格式：**

1. **file-sorter 输出**（推荐）：文件夹内含 `.file-registry.json`，直接读取注册表中的分类信息和文件列表
2. **手动整理的目录**：无注册表时，自动扫描目录结构，按子目录名推断分类

```
📁 张三申请执行李四民间借贷纠纷_整理后/
├── 📁 01_生效文书/
│   ├── 民事判决书_张三诉李四民间借贷纠纷.pdf
│   └── 民事调解书_张三诉王五借款纠纷.pdf
├── 📁 02_执行文书/
│   ├── 执行裁定_终结本次执行程序.pdf
│   └── 网络查控清单_20240115.pdf
├── 📁 03_流水材料/
├── 📁 04_申请文书/
├── 📁 05_主体信息/
├── 📁 06_其他材料/
└── 📄 .file-registry.json
```

## 输出

**主输出文件**：`案件分析底稿_[案件简称].md`，放在案件文件夹根目录。

**辅助输出**：`.case-analysis.json`，机器可读的分析结果，供后续 skill 使用。

## 工作流程

### 第一步：加载材料清单

**1.1 读取注册表**

如存在 `.file-registry.json`，读取：
- `case_name` — 案件名称
- `files[].category` — 文件分类
- `files[].doc_type` — 文书类型
- `files[].parties` — 当事人
- `files[].new_path` — 文件路径

**1.2 无注册表时自扫描**

遍历子目录，生成简易清单，分类规则同 `file-sorter` 第三步。

**1.3 确认案情类型**

根据材料内容判断执行案件子类型：

| 类型 | 关键词特征 | 分析重点 |
|------|-----------|----------|
| 金钱给付 | 借贷/欠款/货款/工程款 | 财产查找、流水追踪 |
| 行为履行 | 腾退/交付/过户/排除妨碍 | 履行可行性、替代方案 |
| 涉公司执行 | 被执行人是公司 | 股东责任、一人公司、关联企业 |
| 终本恢复 | 存在终本裁定 | 新财产线索、股东未实缴出资 |
| 涉夫妻/家庭财产 | 配偶/共有财产 | 夫妻共同债务认定、财产分割 |

### 第二步：提取案件基本信息

**逐份阅读以下核心文书，提取结构化信息：**

#### 2.1 生效文书（01_生效文书/）

**判决书**（优先级最高）：
- 案号（一审案号、二审案号）——从文书首页提取，使用正则 `[（(〔[]\d{4}[）)〕[]].*?号`
- 审理法院
- 原告/被告 → 映射为申请执行人/被执行人
- 案由（如民间借贷纠纷、买卖合同纠纷）
- 立案日期、判决日期
- 判决主文：被告应支付金额（本金）、利息起算日及利率、其他给付义务
- 诉讼费负担
- 是否缺席判决（影响后续执行难度）

**调解书**：
- 调解协议内容
- 分期还款计划（如有时逐期列出）
- 违约责任条款（加速到期条款）

**仲裁裁决书**：
- 仲裁机构
- 裁决内容、履行期限

#### 2.2 执行文书（02_执行文书/）

**执行裁定书**：
- 执行案号
- 执行法院
- 申请执行日期
- 立案日期
- 执行标的（本金 + 利息 + 诉讼费 + 执行费）
- 执行结果类型：
  - `正在执行` — 执行中
  - `终结本次执行程序` — 终本（重点：终本日期、终本理由）
  - `执行完毕` — 已结案
  - `终结执行` — 和解/撤回/其他原因终结
  - `驳回执行申请` — 需分析原因
- 已执行到位的金额

**网络查控清单/查控反馈**：
- 查控日期
- 查控结果摘要（银行/房产/车辆/证券/保险/公积金）
- 实际控制到的财产

**报告财产令及被执行人财产申报**：
- 被执行人自行申报的财产
- 与查控结果的差异（识别隐匿风险）

**限制消费令 / 纳入失信名单决定书**：
- 发布时间
- 是否已解除

**评估报告 / 拍卖公告 / 成交确认书**（如有时）：
- 财产处置进度
- 成交价/流拍价

#### 2.3 申请文书（04_申请文书/）

提取申请人已提交的和待提交的申请类型清单：
- 财产调查类：调查令申请、律师调查令申请
- 查控类：查封申请、冻结申请、续封申请
- 惩戒类：限制消费申请、纳入失信申请、拘留申请、边控申请
- 执行变更类：追加被执行人申请、变更执行主体申请
- 执行异议类：执行行为异议、执行标的异议
- 恢复执行类：恢复执行申请

#### 2.4 主体信息（05_主体信息/）

- 申请执行人：姓名/名称、身份证号/统一社会信用代码、联系方式
- 被执行人：
  - 自然人：姓名、身份证号、住址、联系方式
  - 企业：名称、统一社会信用代码、法定代表人、注册地址、经营状态
- 委托代理人信息（如有）
- 如被执行人为企业，确认是否属于一人有限责任公司（影响后续追加被执行人策略）

#### 2.5 流水材料（03_流水材料/）

- 流水覆盖时间段
- 开户行、账号
- 关键异常交易标记（大额转出、频繁小额、特定对象）

### 第三步：建立事实时间线

**按时间顺序排列所有关键事件，精确到日：**

#### 时间线要素格式

```
| 日期 | 事件 | 类型 | 来源材料 | 页码/行 |
|------|------|------|----------|---------|
| 2023-03-15 | 张三向李四出借100万元 | 债权形成 | 借款合同 | 第1页 |
| 2023-09-15 | 借款到期，李四未还款 | 违约 | 判决书查明事实 | 第3页 |
| 2024-01-10 | 张三类案向XX法院起诉 | 诉讼启动 | 判决书 | 第1页 |
| 2024-06-20 | 一审判决：李四偿还100万元及利息 | 判决 | (2024)XX民初123号判决书 | 全文 |
| 2024-08-01 | 判决生效 | 生效 | 生效证明/判决书送达回证 | — |
| 2024-09-15 | 张三申请强制执行 | 执行立案 | 执行裁定书 | 第1页 |
| 2024-09-20 | 执行法院立案，案号(2024)XX执456号 | 执行立案 | 执行裁定书 | 第1页 |
| 2024-12-10 | 网络查控反馈：无可供执行财产 | 查控 | 网络查控清单 | 全文 |
| 2025-03-15 | 终本裁定：终结本次执行程序 | 终本 | (2024)XX执456号裁定书 | 全文 |
```

#### 时间线建立规则

1. **日期精确度分层**：
   - P0：判决书/裁定书上的日期 → 精确到日
   - P1：合同中约定的日期 → 精确到日
   - P2：材料中提到但未精确标注的 → 标注到月，用 `~` 标记（如 `2024-03~`）
   - P3：无法确定具体时间的 → 标注到阶段，用 `？` 标记

2. **案件状态时间线颜色标记**（在报告中用 emoji 辅助）：
   - 🔵 债权形成阶段
   - 🟡 诉讼阶段
   - 🟢 执行阶段
   - 🔴 终本/停滞
   - ⚪ 待补充时间

3. **关键时间节点校验**：
   - 判决生效日 → 申请执行日 是否超过 2 年（申请执行时效）
   - 终本日期 → 当前日期 间隔（恢复执行时机）
   - 查封/冻结日期 → 到期日期（续封预警）
   - 各阶段的时间间隔是否异常（警惕程序拖延）

### 第四步：财产线索全维度盘点

#### 4.1 财产线索来源清单

| 财产类型 | 线索来源 | 线索状态 | 详情 |
|---------|---------|---------|------|
| 不动产 | 查控反馈/当事人提供/公开信息 | 已查控/待查证/未发现 | 坐落、产权证号、是否首封 |
| 银行存款 | 查控反馈/流水材料 | 已冻结/已扣划/余额不足 | 开户行、账号、冻结金额 |
| 车辆 | 查控反馈/当事人提供 | 已查封/未实际控制/未发现 | 车牌号、型号、是否已扣押 |
| 股权/投资 | 查控反馈/工商信息 | 已冻结/待查证 | 公司名称、持股比例、出资额 |
| 到期债权 | 材料中线索 | 待核实 | 次债务人名称、债权金额 |
| 公积金 | 查控反馈 | 已冻结/待查证 | 缴存机构、余额 |
| 保险/理财 | 查控反馈 | 已冻结/待查证 | 保险类型、现金价值 |
| 知识产权 | 公开信息 | 待查证 | 商标/专利/著作权 |
| 应收账款 | 合同/交易记录 | 待核实 | 付款方、金额、到期日 |

#### 4.2 隐匿财产手段识别

根据材料中的线索，识别被执行人可能使用的隐匿手段：

| 隐匿手段 | 识别信号 | 应对方向 |
|---------|---------|---------|
| 离婚转移财产 | 执行前突击离婚；财产全归配偶 | 调取离婚协议；债权人撤销权之诉 |
| 低价/无偿转让 | 判决前后的房产过户、股权变更记录 | 债权人撤销权之诉（注意1年/5年除斥期间） |
| 关联企业走账 | 流水显示与关联公司频繁大额往来 | 调取关联企业工商档案；法人人格否认 |
| 虚假债务 | 执行期间突然出现其他债权人 | 审查债务真实性；分配方案异议 |
| 现金交易 | 无可查银行流水；经营场所仍在运营 | 现场调查；微信/支付宝流水 |
| 借名持有 | 实际使用但非名下房产/车辆/账户 | 实地调查；间接证据链 |
| 恶意注销公司 | 被执行企业已注销 | 调取工商注销档案；追究清算责任 |
| 变更法定代表人 | 执行立案后突击变更 | 原法定代表人限消；证明实际控制 |

### 第五步：材料缺口分析

**逐项比对"执行案件理想材料清单"与现有材料，生成缺口清单：**

#### 5.1 必备材料检查

| 材料 | 已有 | 说明 |
|------|------|------|
| 生效判决书/调解书/仲裁裁决 | ✅ / ❌ | |
| 生效证明（如判决书未注明生效日期） | ✅ / ❌ | 如缺：需联系审理法官开具 |
| 执行裁定书（如已申请执行） | ✅ / ❌ | |
| 申请执行人身份证明 | ✅ / ❌ | |
| 被执行人身份信息 | ✅ / ❌ | 自然人：身份证复印件；企业：工商信息 |
| 被执行人行财产线索清单 | ✅ / ❌ | 如缺：口头询问当事人，记录线索 |
| 授权委托书（如代理） | ✅ / ❌ | |

#### 5.2 补强材料缺口

按优先级列出应调取/补充的材料：

```
🔴 紧急（影响执行进度）
- [ ] 被执行人最新工商内档（含股东身份信息、出资情况、变更记录）
  → 去哪里查：XX市市场监督管理局 / XX区政务服务中心
  → 需要材料：调查令/介绍信/执业证/授权委托书
- [ ] 被执行人银行流水近3年（XX银行 XX支行）
  → 需要材料：法院调查令

🟡 重要（影响执行策略）
- [ ] 被执行人婚姻登记信息（核查是否离婚逃债）
  → 去哪里查：XX市民政局
- [ ] 被执行人关联企业工商信息
  → 可通过企查查/天眼查初步查询

🟢 一般（优化执行方案）
- [ ] 被执行人社保缴纳记录
- [ ] 被执行人公积金缴存信息
```

#### 5.3 申请人待提供材料

列出需要客户（申请执行人）补充提供的材料：
- 被执行人可能的住址/工作单位
- 被执行人的联系方式（手机号、微信）
- 与被执行人的其他往来记录（微信聊天、转账记录等）
- 被执行人的生活习惯/出行规律/消费场所

### 第六步：输出案件分析底稿

## 特殊处理规则

### 多被执行人案件

如案件涉及多个被执行人，为每个被执行人分别建立：
- 财产线索表
- 隐匿手段分析
- 主体信息

在基本信息表中列出所有被执行人，用表格区分。

### 涉公司被执行人

如被执行人为企业，额外分析：
1. **股东出资情况**：注册资本、实缴情况、认缴期限
   - 如认缴期限未届满 → 是否可适用出资加速到期
   - 如已转让股权且未实缴 → 原股东在未出资范围内承担责任
2. **一人公司风险**：如被执行人为一人有限责任公司
   - 股东不能证明财产独立的 → 对公司债务承担连带责任
3. **关联企业**：是否存在关联企业财产混同
4. **注销/吊销**：是否已注销或吊销
   - 注销 → 追究清算责任
   - 吊销 → 仍可执行，但需注意时效

### 终本案件特殊处理

如为终本案件（存在终本裁定），额外分析：
1. 终本前已采取的查控措施及到期情况
2. 终本后是否出现新财产线索
3. 恢复执行的前置条件是否满足
4. 终本期间的利息是否仍在计算

### 日期无法确定时

- 不强行推算、不编造日期
- 标注 `待确认`，列出推断范围
- 提示用户从哪些渠道可以确认该日期

### 信息矛盾时

- 如不同材料对同一事实的记载不一致
- 标注矛盾点，优先采信法院文书
- 在报告中用 `⚠️ 矛盾提示` 标记

## 执行约束

1. **不编造**：所有分析结论必须有材料支撑，不得推断无依据的事实
2. **谨慎标注**：不确定的信息统一标注"待确认"、"初步判断"
3. **法律意见边界**：本分析是事实梳理和策略建议，不构成正式法律意见
4. **日期严格**：时间线中的日期必须能在材料中找到出处
