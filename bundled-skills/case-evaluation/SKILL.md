# 案件评估 Skill（"这个案子值不值得做"）

## 功能概述

执业律师接执行案件前，需要回答三个核心问题：

1. **钱在哪** —— 有没有可执行的财产？
2. **钱能回来吗** —— 回款概率有多大？要多久？
3. **值得做吗** —— 投入多少时间？怎么收费才不亏？

这三个问题的答案分别散落在案情分析、财产调查、流水分析的结果里。本 skill 把它们聚合起来，做减法——去掉已经穷尽的死线索，聚焦可行动的活线索，量化变现可能性和工作量，最终输出"接不接、怎么接"的决策建议。

**内部用途**（团队评估是否接案）和**外部用途**（向客户汇报案件推进方案与回款预期）都适用。

## 使用场景

- 新客户咨询，评估是否接受执行案件委托
- 终本案件，评估是否有恢复执行价值
- 多个备选案件，需要排优先级
- 向客户汇报：目前查到什么、接下来做什么、预计多久能回款
- 团队内部讨论：这个案子要投多少律师时间，按什么收费模式

## 输入

**聚合上游三个 report 的数据文件：**

```
输入源：
├── .case-analysis.json       ← case-analysis 输出（事实时间线、隐匿分析、材料缺口）
├── .execution-survey.json    ← execution-survey 输出（财产线索、司法风险、关联企业）
└── .flow-analysis.json       ← wechat-flow-analysis 输出（如有，流水异常+线索）
```

> 至少需要前两个文件。流水分析可选，但强烈推荐——流水往往是最好的财产线索来源。

## 工作流程

### 第一步：资产清单去重与分类

**1.1 汇总所有财产线索**

从三个上游文件汇总全部已发现和待查证的财产线索：

| 来源 | 提取字段 |
|------|---------|
| `.case-analysis.json` → `property_clues` | 从材料中直接发现的线索 |
| `.execution-survey.json` → `asset_clues` | 企查查查到的资产（抵押/知产/投资/拍卖） |
| `.flow-analysis.json` → `property_clues` | 从流水中推断的线索（房产/车辆/账户/经营） |

**1.2 去重**

同一财产线索可能被多个 skill 识别到，合并去重：
- 同一地址的房产 → 合并
- 同一车牌号的车辆 → 合并
- 同一银行卡号 → 合并

**1.3 按可执行性分类**

| 类别 | 定义 | 评估逻辑 |
|------|------|---------|
| **A 类**：已查明的可执行财产 | 已知财产类型+估值+查控状态 | 直接进入变现可行性评估 |
| **B 类**：有明确查证路径的线索 | 知道去哪里查（如"去X市不动产中心查XX小区某户"） | 评估查证成本 |
| **C 类**：模糊线索 | 知道可能有但不知道在哪（如"可能有车"） | 标注不确定性，影响回款概率 |
| **D 类**：无财产线索 | 三份报告均未发现 | 结论：目前无可执行财产 |

--- 输出阶段提示 ---
```
🏠 资产清单汇总（去重后）：
- A 类（已查明可执行）：[X] 项，预估总值 [XXX 万元]
- B 类（有明确查证路径）：[X] 项
- C 类（模糊线索）：[X] 项
- D 类（无）：[是/否]
```

### 第二步：财产变现可行性评估

对 A 类和 B 类线索逐项评估变现难度。

#### 2.1 不动产

| 评估维度 | 判断标准 | 评分 |
|---------|---------|------|
| 查封顺位 | 首封=5分，轮候第2=3分，轮候第3+=1分 | /5 |
| 估值与债务比 | 估值>债务=5分，估值≈债务=3分，估值<债务=1分 | /5 |
| 抵押负担 | 无抵押=5分，有抵押但余值够=3分，抵押>估值=1分 | /5 |
| 占有人 | 被执行自住=3分（腾退难），空置=5分，出租但有租约=2分 | /5 |
| 唯一住房 | 否=5分，是=1分（需提供安置方案） | /5 |
| **综合变现难度** | ≥20分=低，15-19分=中，≤14分=高 | — |

#### 2.2 车辆

| 评估维度 | 判断标准 | 评分 |
|---------|---------|------|
| 查封状态 | 已查封+已实际扣押=5分，已查封未扣押=2分，未查封=1分 | /5 |
| 车龄 | <3年=5分，3-6年=3分，>6年=1分 | /5 |
| 估值 | ≥10万=5分，5-10万=3分，<5万=1分 | /5 |
| **综合变现难度** | ≥12分=低，8-11分=中，≤7分=高 | — |

#### 2.3 银行存款

| 评估维度 | 判断标准 | 评分 |
|---------|---------|------|
| 冻结状态 | 已冻结+余额足够=5分，已冻结但余额不足=2分，未冻结=1分 | /5 |
| 余额 | ≥债务50%=5分，10-50%=3分，<10%=1分 | /5 |
| **综合变现难度** | ≥8分=低，5-7分=中，≤4分=高 | — |

#### 2.4 股权/对外投资

| 评估维度 | 判断标准 | 评分 |
|---------|---------|------|
| 被投资企业经营状态 | 正常经营+盈利=5分，正常经营但盈亏不明=3分，停业/吊销=1分 | /5 |
| 持股比例 | ≥50%（控股）=5分，20-50%=3分，<20%=1分 | /5 |
| 冻结状态 | 已冻结=5分，未冻结=1分 | /5 |
| 估值难度 | 有评估报告=5分，可委托评估=3分，难以估值=1分 | /5 |
| **综合变现难度** | ≥16分=低，11-15分=中，≤10分=高 | — |

#### 2.5 知识产权

- 商标：需评估市场价值（知名度/续展状态/是否实际使用）
- 专利：需评估技术价值和剩余保护期
- 变现难度默认为"高"（知识产权拍卖成交率很低）——除非为知名商标或核心专利

#### 2.6 到期债权/应收账款

- 需核实次债务人的偿债能力
- 如需通过代位权诉讼实现 → 增加 6-12 个月周期
- 变现难度：中到高

--- 输出阶段提示 ---
```
💰 变现可行性汇总：
| 财产 | 类型 | 估值 | 变现难度 | 预计回收 | 预计周期 |
|------|------|------|---------|---------|---------|
| XX房产 | 不动产 | 200万 | 低 | 160-200万 | 3-6月 |
| XX股权 | 投资 | 不明 | 高 | 0-50万 | 12月+ |
```

### 第三步：偿债意愿与能力评估

**3.1 偿债意愿判断**

| 维度 | 信号 | 评分（-2 ~ +2） |
|------|------|----------------|
| 沟通状态 | 正常联系并配合=+2，偶尔联系=0，失联=-2 | |
| 履行历史 | 已部分履行=+2，口头承诺未兑现=0，从未履行=-2 | |
| 财产申报 | 如实申报=+2，少报/瞒报=-1，拒报=-2 | |
| 失信行为 | 无失信=-2（加分项），有失信=-2（减分项）→ 反向 | |
| 限消违规 | 未发现违规=0，有违规记录=-1，流水中有违规证据=-2 | |
| **意愿评分** | ≥+2=高，-1到+1=中，≤-2=低 | |

**3.2 偿债能力判断**

| 维度 | 信号 | 评分 |
|------|------|------|
| 已查明资产足以覆盖债务 | 是=+2，部分覆盖=+1，无资产=-2 | |
| 有持续收入 | 流水中有稳定工资/经营收入=+2，偶尔收入=0，无收入=-2 | |
| 年龄/健康 | 中青年+健康=+1（有未来收入），年迈/重病=0 | |

**3.3 四象限评估**

将意愿和能力交叉，确定案件类型：

```
              偿债能力
            强          弱
    ┌──────────┬──────────┐
强   │ A 类     │ C 类     │
    │ 理想案件  │ 需寻找   │
    │ 重点推进  │ 收入来源  │
偿  ├──────────┼──────────┤
债  │ B 类     │ D 类     │
意  │ 典型"老赖"│ 最难的   │
愿  │ 以惩戒驱动│ 案子     │
弱   └──────────┴──────────┘
```

- **A 类（能力强+意愿强）**：最优，推动协商和解
- **B 类（能力弱+意愿强）**：协助分期还款方案
- **C 类（能力强+意愿弱）**：执行律师的主战场，以查控+惩戒+拒执罪组合施压
- **D 类（能力弱+意愿弱）**：坦诚告知客户回款前景

--- 输出阶段提示 ---
```
🎯 偿债画像：
- 意愿：[高/中/低]
- 能力：[强/中/弱]
- 案件类型：[A/B/C/D]
- 策略方向：[协商/强制/组合施压/现实预期]
```

### 第四步：回款概率评估

**4.1 分级评估逻辑**

| 回款概率 | 条件 |
|---------|------|
| **70%+（高）** | A 类财产足以覆盖债务 + 已首封 + 变现难度低 + 意愿强 |
| **50-70%（中高）** | A 类财产部分覆盖 + B 类线索可查证 + 变现难度中 |
| **30-50%（中）** | A 类财产较少 + B/C 类线索为主 + 需衍生诉讼 |
| **10-30%（中低）** | 仅C类线索 + 被执行人失联/意愿低 + 需大量调查 |
| **<10%（低）** | 无任何财产线索 + 被执行人失联/已注销 |

**4.2 双情景预估**

| 情景 | 假设条件 | 回款概率 | 预计回款周期 |
|------|---------|---------|------------|
| 保守 | 仅变现A类财产，B/C类均未查实 | [XX%] | [XX-XX月] |
| 乐观 | A类变现+B类部分查实+拒执罪施压见效 | [XX%] | [XX-XX月] |

**4.3 回款金额区间**

| 情景 | 预计回收金额 | 占执行标的比例 |
|------|------------|-------------|
| 保守 | [XXX万] | [XX%] |
| 乐观 | [XXX万] | [XX%] |

### 第五步：工作量估算

**5.1 分模块律师时间估算**

| 工作模块 | 律师时间 | AI替代比例 | 说明 |
|---------|---------|-----------|------|
| 材料整理与索引 | 1-2h | 80% (file-sorter) | 人工复核分类结果 |
| 案情分析与底稿 | 2-3h | 70% (case-analysis) | 核实关键信息+策略判断 |
| 财产调查（线上） | 2-3h | 60% (execution-survey) | AI查工商/司法风险，律师分析 |
| 流水分析 | 3-5h | 60% (wechat-flow-analysis) | AI标记异常，律师核实+判断 |
| 查档（实地） | 4-8h/次 | 10%（不可替代） | 交通+等待+翻阅+拍照 |
| 申请书起草 | 2-3h/份 | 70% (application-generator) | AI生成草稿，律师修订 |
| 法院沟通 | 8-16h | 0%（不可替代） | 电话/约见法官/递交材料/跟进 |
| 现场调查 | 4-8h/次 | 0%（不可替代） | 实地查看/拍照/走访 |
| 客户沟通 | 3-5h | 0%（不可替代） | 汇报+答复+协调 |
| 案卷归档 | 1-2h | 90% | — |
| **合计（首执案件，3-6月周期）** | **30-55h** | | 纯律师时间约 18-35h |
| **合计（终本恢复，6-12月周期）** | **50-80h** | | 纯律师时间约 30-55h |

**5.2 关键节点时间线**

```
接受委托
  ↓ 1-2周
材料整理+案情分析
  ↓ 2-4周
财产调查（线上+线下）
  ↓ 1-2周
查档（工商/不动产/婚姻）
  ↓ 2-4周
提交财产线索/查控申请
  ↓ 1-3月
法院查控反馈
  ↓ 2-6月
评估拍卖（如有不动产）
  ↓ 1-3月
案款分配 → 回款
```

### 第六步：代理建议

**6.1 是否建议接受委托**

| 结论 | 条件 |
|------|------|
| ✅ **建议接受** | 有A类资产+变现难度低+意愿尚可+回款概率≥50% |
| 🟡 **谨慎接受** | 仅有B/C类线索+需大量调查+回款30-50% |
| ⚠️ **有条件接受** | 线索有限但客户愿意投入调查成本+接受不确定性 |
| ❌ **不建议接受** | 无任何线索+被执行人失联/已注销/明显无财产 |

**6.2 收费模式建议**

| 模式 | 适用场景 | 建议比例/金额 |
|------|---------|-------------|
| **半风险**（基础+提成） | 有明确财产线索+回款概率中高 | 基础费：市场行情的50-70%<br>提成：回款额的10-20% |
| **全风险** | 线索较多但变现不确定+客户预算有限 | 提成：回款额的20-30% |
| **固定收费** | 仅做调查和方案（不含执行实施）<br>回款前景不乐观，客户明确"先查查看" | 按模块计费 |
| **计时收费** | 案件复杂+不确定因素多<br>机构客户 | 按小时费率 |

**6.3 风险提示**

```
⚠️ 需向客户明确告知的风险：

1. 财产处置风险：[如唯一住房无法处置 / 轮候查封需等待首封处置后分配]
2. 衍生诉讼风险：[如需撤销权/代位权之诉，将增加XX月周期和XX元诉讼成本]
3. 参与分配风险：[如其他债权人参与分配，回款比例将下降]
4. 终本风险：[如查控反馈无可供执行财产，案件可能再次终本]
5. 时间风险：[预计回款周期XX月，实际可能因法院进度、被执行人不配合等因素延长]
```

## 与前后 Skill 的协作

`case-evaluation` 是本套 skill 的**决策汇聚点**——上游三个分析 skill 的输出在此汇聚，产出"接不接/怎么收/做哪些"的核心判断。同时它也是**客户沟通的起点**——评估报告可以直接作为向客户汇报的底稿。

```
case-analysis ──┐
execution-survey ──┼──→ case-evaluation ──→ 客户沟通 / 团队决策
wechat-flow-analysis ┘         │
                               └──→ application-generator（下一步行动→申请书）
                               └──→ enforcement-tracker（期限→提醒）
```
