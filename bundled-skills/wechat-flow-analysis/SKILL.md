# 微信流水分析 Skill

## 功能概述

微信流水是执行律师的"金矿"——里面藏着被执行人的真实消费能力、资金去向、隐藏财产和活动轨迹。但原始流水文件动辄成千上万行，金额单位还是"分"，直接读等于大海捞针。

本 skill 做四件事：**格式标准化**（分转元、字段对齐）、**全景统计**（流入/流出/余额趋势）、**多维异常识别**（大额转移/高消费违规/单独清偿/隐匿收入/循环转账）、**线索提取**（财产线索/活动范围/关联方）。

**最终输出不是数据罗列——是按"哪些交易可能导致拘留/哪些线索可以查封财产/哪些行为构成拒执罪"的逻辑组织过的分析报告。**

## 使用场景

- 拿到法院调取的微信流水光盘，需要快速发现有价值的线索
- 判断被执行人是否违反限制消费令
- 寻找被执行人资金去向（可能转移给了谁）
- 发现被执行人的实际经营收入（虽然账户可能不是他名字）
- 为拒执罪控告收集证据
- 评估被执行人的真实偿债能力

## 输入

用户指定微信流水文件的路径。

**支持的流水格式：**

1. **微信司法调取流水**（法院调取的原始格式）——金额单位为"分"
2. **微信个人导出流水**（微信APP导出的账单）——金额单位通常为"元"
3. **微信商户流水**（商户号后台导出）
4. **支付宝流水**（兼容，同样支持）

> 如不确定格式，skill 会自动检测表头并适配。

## 工作流程

### 第一步：读取流水并标准化

**1.1 自动检测文件格式**

```python
# 检测表头关键词，判断流水来源和格式
微信司法调取: "交易时间" "交易类型" "交易对方" "商品" "收/支" "金额(分)" "支付方式" "当前状态" "交易单号" "商户单号"
微信个人导出: "交易时间" "交易类型" "交易对方" "商品" "金额(元)" "支付方式" "当前状态"
微信商户流水: "交易时间" "商户号" "交易单号" "交易金额(元)" "手续费"
支付宝流水: "交易时间" "交易对方" "商品说明" "金额" "收/支" "交易状态"
```

**1.2 金额单位转换**

| 流水来源 | 原始单位 | 转换 |
|---------|---------|------|
| 微信司法调取 | **分** | ÷ 100 = 元 |
| 微信个人导出 | 元 | 无需转换 |
| 微信商户 | 元 | 无需转换 |
| 支付宝 | 元 | 无需转换 |

> ⚠️ **关键规则**：微信司法调取的流水金额单位是"分"！如看到一条交易金额为 `500000`，实际是 `5,000.00 元`。未转换直接分析会导致数据完全错误。

**1.3 结构化字段**

统一转换为以下标准字段：

| 标准字段 | 说明 | 来源映射 |
|---------|------|---------|
| `date` | 交易日期 | `交易时间` |
| `time` | 交易时间 | 从 `交易时间` 分离 |
| `type` | 收/支 | `收/支` 列，标准化为 `收入` / `支出` |
| `amount` | 金额（元） | 分转元后的数值 |
| `balance` | 余额（元） | 如流水提供 |
| `counterparty` | 交易对手 | `交易对方` |
| `counterparty_account` | 对方账号 | 如有 |
| `description` | 交易摘要/商品描述 | `商品` / `商品说明` |
| `method` | 支付方式 | `支付方式`（零钱/零钱通/银行卡） |
| `status` | 交易状态 | `当前状态` |
| `transaction_id` | 交易单号 | `交易单号` |
| `merchant_id` | 商户单号 | `商户单号` |

**1.4 数据清洗**

- 排除重复行（同一交易单号出现两次）
- 排除测试交易（金额为 0.01 元的验证交易）
- 标记 `status` 为"已退款""退款成功"的交易（不算实际收支）
- 记录数据异常（如某行缺少日期、金额为负数等）

--- 输出阶段提示 ---
```
📊 流水数据概览：
- 文件来源：[微信司法调取 / 微信个人导出 / 支付宝]
- 数据行数：12,345 行
- 时间跨度：2023-01-01 至 2025-12-31
- 金额单位：已从"分"转换为"元"（如需）
- 数据异常：[X 行缺少日期 / X 行金额异常]
```

### 第二步：基础统计

**2.1 整体资金概览**

| 统计维度 | 计算方式 |
|---------|---------|
| 期间总流入 | SUM(收入金额) |
| 期间总流出 | SUM(支出金额) |
| 净流入 | 总流入 - 总流出 |
| 期初余额 | 第一笔交易前的余额 |
| 期末余额 | 最后一笔交易后的余额 |
| 日均余额 | 每日余额的平均值 |
| 日均交易笔数 | 总交易笔数 ÷ 天数 |
| 笔均金额 | 总交易金额 ÷ 交易笔数 |

**2.2 按年度统计**

| 年度 | 总流入 | 总流出 | 净流入 | 期末余额 | 交易笔数 |
|------|--------|--------|--------|----------|----------|
| 2023 | | | | | |
| 2024 | | | | | |
| 2025 | | | | | |

**2.3 按月度趋势**

逐月统计流入/流出，生成趋势描述（不是图表，而是文字总结）。

**2.4 收入来源分布**

按交易对手归类，统计 Top 10 收入来源：

| 排名 | 交易对手 | 收入笔数 | 收入总额 | 占比 | 交易类型 |
|------|---------|---------|---------|------|---------|
| 1 | XX公司 | 24 | 120,000 | 22% | 工资/劳务 |
| 2 | 张三 | 15 | 80,000 | 15% | 转账 |

**2.5 支出去向分布**

按交易对手/交易类型归类，统计 Top 10 支出：

| 排名 | 交易对手 | 支出笔数 | 支出总额 | 占比 | 交易类型 |
|------|---------|---------|---------|------|---------|

--- 输出阶段提示 ---
```
💰 资金概览：
- XXXX年度至XXXX年度
- 总流入：X,XXX,XXX 元
- 总流出：X,XXX,XXX 元
- 净流入：XXX,XXX 元
- 期末余额：XXX,XXX 元
```

### 第三步：大额交易筛选

**3.1 大额交易定义**

| 级别 | 金额阈值 | 标记颜色 |
|------|---------|---------|
| 超大额 | ≥ 50,000 元 | 🔴 |
| 大额 | ≥ 5,000 元（默认） | 🟡 |
| 中等 | ≥ 1,000 元 | 🟢 |

> 阈值可根据执行标的调整。标的额 100 万以上的案件，建议将"大额"调至 10,000 元。

**3.2 大额交易清单**

按金额降序排列：

| 日期 | 收支 | 金额 | 交易对手 | 摘要 | 交易类型 | 标记 |
|------|------|------|----------|------|----------|------|
| 2024-03-15 | 支出 | -500,000 | 王五 | 转账 | 转账 | 🔴 |
| 2024-06-20 | 收入 | +200,000 | XX公司 | 货款 | 经营 | 🟡 |

**3.3 大额交易汇总**

- 大额交易总笔数 / 总金额
- 大额交易占总交易金额的比例
- 大额支出的主要去向
- 大额收入的主要来源

### 第四步：异常交易多维识别

**这是整个分析的核心——从交易海洋里捞出"能用的"证据。**

#### 4.1 🔴 疑似资产转移

**识别规则：**

| 规则 | 检测逻辑 | 执行意义 |
|------|---------|---------|
| 判决/执行立案后大额转出 | 筛选 `date > 判决日期` 的 ≥ 10,000 元支出 | 可能构成拒执罪 |
| 关联方大额转账 | 交易对手与被执行人同姓/同一地址/关联企业 | 转移至亲友/关联方 |
| 整数金额转账 | 金额为整千/整万的转账（如 50,000.00） | 非正常消费，更像转移 |
| 备注可疑的转账 | 摘要含"借款""还款""投资""买房""还贷" | 资金有指向性去向 |
| 短期内资金快进快出 | 24 小时内收入随即等额/近似额支出 | 过桥/洗钱/虚假交易 |

**输出格式：**

```
🔴 疑似资产转移

| 日期 | 金额 | 交易对手 | 摘要 | 可疑原因 |
|------|------|----------|------|----------|
| 2024-03-15 | -500,000 | 王五（同姓） | "借款" | 执行立案后转出，收款人与被执行人同姓，备注"借款"但无借据佐证 |
```

#### 4.2 🔴 疑似单独清偿其他债务

**识别规则：**

| 规则 | 检测逻辑 | 执行意义 |
|------|---------|---------|
| 固定日期定额转账 | 每月/每周固定日期向同一对象转相近金额 | 疑似偿还其他个人债务（未经生效判决确认的债权） |
| 备注含"还""利息""还款" | 摘要中的关键词 | 直接承认还款 |
| 向非金融机构/非银行的大额支付 | 交易对手不是银行/正规金融机构 | 可能偿还民间借贷 |

> ⚠️ **执行意义**：被执行人在执行期间向特定债权人单独清偿，可能构成《最高人民法院关于适用〈中华人民共和国民事诉讼法〉的解释》第511条所规制的偏颇清偿，申请执行人可主张撤销或提出分配方案异议。

**输出格式：**

```
🔴 疑似单独清偿其他债务

| 日期 | 金额 | 收款人 | 规律 | 备注 |
|------|------|--------|------|------|
| 每月15日 | -10,000 | 张三 | 连续12个月，每月固定日期 | 无"张三"作为债权人的判决书 |
```

#### 4.3 🟡 循环转账（借钱还债模式）

**识别规则：**

| 规则 | 检测逻辑 |
|------|---------|
| 先收后付 | 收到一笔大额转账后，1-3天内转出相近金额 |
| 金额接近 | 收入金额与后续支出金额差异 < 5% |
| 模式重复 | 相同模式出现 2 次以上 |

**输出格式：**

```
🟡 疑似循环转账（借钱还债）

| 日期 | 收入 | 来源 | → 日期 | 支出 | 去向 | 差额 |
|------|------|------|--------|------|------|------|
| 03-01 | +100,000 | 李四 | → 03-03 | -98,500 | 王五 | 1,500 |
```

#### 4.5 🟢 疑似财产线索

从流水交易记录中反向推断被执行人的财产：

| 线索类型 | 检测方式 | 执行动作 |
|---------|---------|---------|
| 房产 | 物业费/水电费/燃气费/房租支付记录 → 锁定小区 | 去不动产登记中心查该地址产权 |
| 车辆 | 停车费/加油费/充电费/高速费/洗车费/保养费 → 推断有车 | 去车管所查该车牌号 |
| 银行卡 | 向同一银行卡转入资金（还款/理财）→ 发现隐藏账户 | 申请法院调取该银行卡流水 |
| 经营实体 | 频繁收款来自同一客户群 → 推断经营业务 | 现场调查经营场所 |
