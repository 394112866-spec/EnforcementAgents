# 文件整理 Skill（执行案件材料智能分类）

## 功能概述

客户发来的案件材料通常混乱不堪——文件名是微信自动生成的乱码、多份判决书和执行裁定混在一起、银行流水和身份证明掺杂其中。本 skill 解决"拿到材料先整理"这第一步。

**核心原则：原始文件只读，不修改不删除；整理结果输出到新目录。**

## 使用场景

- 新案件委托，客户一次性发来大量材料
- 执行过程中客户补交新材料，需要合并到已有案件目录
- 案件材料混杂，需要快速建立可检索的规范档案

## 输入

用户指定一个文件夹路径（或直接拖入/粘贴路径），文件夹内包含客户发来的原始材料文件。

支持的文件格式：
- PDF（`.pdf`）
- Word（`.docx`、`.doc`）
- 图片（`.jpg`、`.jpeg`、`.png`、`.bmp`、`.tiff`）
- Excel（`.xlsx`、`.xls`、`.csv`）
- 压缩包（`.zip`、`.rar`）——先解压再分类
- 文本（`.txt`）

## 输出

```
📁 [案件名]_整理后/
├── 📁 01_生效文书/          # 判决书、调解书、仲裁裁决书
├── 📁 02_执行文书/          # 执行裁定、查控清单、协助执行通知书等
├── 📁 03_流水材料/          # 银行流水、微信流水、支付宝流水
├── 📁 04_申请文书/          # 调查令申请、查封/冻结申请、恢复执行申请等
├── 📁 05_主体信息/          # 身份证、营业执照、工商信息截图
├── 📁 06_其他材料/          # 无法明确归类但有价值的材料
├── 📁 待确认/               # 无法识别类型的文件
└── 📄 _文件映射表.md        # 原始路径 → 新路径的对照表
```

## 工作流程

### 第一步：环境准备

1. 确认用户指定的源文件夹路径存在且可读
2. 读取用户偏好配置（如 `config/user-preferences.json` 存在）：
   - `output_dir`：输出目录，默认在源文件夹同级创建 `[源文件夹名]_整理后`
   - `case_name`：案件名称（用于文件命名），未配置时从材料中自动推断
3. 创建输出目录结构（上述 7 个目录 + 待确认）

### 第二步：扫描与识别

遍历源文件夹内所有文件（含子文件夹），对每个文件：

**a) 判断文件类型**

| 扩展名 | 类型标记 | 处理方式 |
|--------|---------|---------|
| `.pdf` | PDF | 直接读取文本层（如无可提文字层则 OCR 首页） |
| `.docx` / `.doc` | Word | 使用 docx skill 或直接读取文本 |
| `.jpg` / `.jpeg` / `.png` / `.bmp` / `.tiff` | 图片 | OCR 识别文字内容 |
| `.xlsx` / `.xls` / `.csv` | Excel | 读取表头与关键行 |
| `.zip` / `.rar` | 压缩包 | 解压到临时目录，递归处理其中文件 |
| `.txt` | 文本 | 直接读取 |

**b) 文本内容提取优先级**

1. PDF 有文字层 → 提取首页内容（前 500 字）
2. PDF 无文字层 → 跳过（不在此 skill 中做 OCR，标记为需 OCR）
3. Word → 提取标题 + 前 500 字
4. 图片 → 跳过内容提取，仅通过文件名关键词判断
5. Excel → 提取表头 + Sheet 名称
6. 无法提取内容的文件 → 仅通过文件名判断

### 第三步：分类判定

对每个文件，按以下优先级判定归属类别：

#### 判定规则表

| 优先级 | 命中条件 | 归类到 |
|--------|---------|--------|
| 1 | 文件名/内容含"判决书"、"民事判决"、"刑事判决"、"行政判决" | `01_生效文书/` |
| 1 | 文件名/内容含"调解书"、"民事调解"、"调解协议"、"仲裁裁决" | `01_生效文书/` |
| 2 | 文件名/内容含"执行裁定"、"执行通知书"、"报告财产令"、"限制消费令"、"纳入失信名单"、"网络查控"、"查控清单"、"协助执行"、"执行完毕"、"终结本次"、"终本"、"终结执行"、"执行异议"、"询价"、"评估报告"、"拍卖"、"变卖"、"以物抵债"、"到账"、"案款" | `02_执行文书/` |
| 3 | 文件名/内容含"银行流水"、"交易明细"、"对账单"、"微信流水"、"支付宝流水"、"转账记录"、"收支明细" | `03_流水材料/` |
| 4 | 文件名/内容含"申请书"、"调查令"、"调查令申请"、"查封申请"、"冻结申请"、"恢复执行申请"、"执行异议申请"、"复议申请"、"追加被执行人申请"、"变更执行主体申请"、"悬赏执行申请"、"拘留申请"、"边控申请"、"限制出境申请"、"审计申请" | `04_申请文书/` |
| 5 | 文件名/内容含"身份证"、"营业执照"、"统一社会信用代码"、"法定代表人"、"法人身份证明"、"授权委托书"、"律师事务所函"、"指派函"、"地址确认"、"送达地址" | `05_主体信息/` |
| 99 | 以上均不匹配 | `待确认/` |

#### 判定增强

- **多关键词命中**：如某文件同时匹配"判决书"和"执行"，以最先出现的关键词为准
- **Excel 文件**：表头含"交易日期""借方金额""贷方金额""余额"等关键词 → 直接归入 `03_流水材料/`
- **命名混乱的多页文件**：如同一份判决书的扫描件拆成多个图片 → 识别后合并为一份 PDF，统一命名
- **文件名完全无法识别**（如 `mmexport1632123456.jpg`）→ 尝试 OCR/内容提取后判定；仍无法判定 → `待确认/`

### 第四步：文件重命名

#### 命名格式

```
[文件类型]_[当事人/关键词]_[补充说明].[后缀]
```

#### 文件类型标准化

| 原始类型 | 标准化名称 |
|---------|-----------|
| 判决书 | `民事判决书` / `刑事判决书` / `行政判决书` |
| 调解书 | `民事调解书` |
| 执行裁定 | 保留原始裁定类型，如 `执行裁定_冻结银行存款` |
| 银行流水 | `银行流水` / `微信流水` / `支付宝流水` |
| 申请书 | 保留申请类型，如 `申请书_调查令` / `申请书_恢复执行` |
| 身份证明 | `身份证` / `营业执照` / `授权委托书` |

#### 重命名示例

| 原文件名 | 新文件名 |
|---------|---------|
| `微信图片_20250101123456.jpg` | `民事判决书_张三诉李四民间借贷纠纷.pdf` |
| `mmexport1632123456.jpg` | `民事判决书_第3页.pdf` |
| `流水.xlsx` | `银行流水_李四_中国银行2024年度交易明细.xlsx` |
| `doc001.pdf` | `执行裁定_查封被执行人李四名下房产.pdf` |
| `IMG_20240315_101010.jpg` | `身份证_被执行人李四_正面.jpg` |

#### 重命名规则

1. **必须保留原始扩展名**，不得改变文件格式
2. 如提取不到当事人名称，用 `未知当事人` 占位
3. 如前缀重复，自动追加序号：`民事判决书_张三诉李四民间借贷纠纷_02.pdf`
4. 清除文件名中的非法字符：`< > : " | ? * \ /` → 替换为 `_`
5. 文件名总长度不超过 200 字符（含扩展名），超长时从关键词部分截断

## 特殊处理规则

### 压缩包处理

1. 检测到 `.zip` / `.rar` → 使用 `unzip` 或相应工具解压到临时目录
2. 递归处理解压后的文件（同样走第二步~第四步）
3. 处理完成后清理临时目录
4. 在文件映射表中标注来源压缩包

### 多页扫描件处理

1. 检测连续命名的图片文件（如 `IMG_001.jpg`、`IMG_002.jpg`、`IMG_003.jpg`）
2. 如后续几页都有同样的"判决书"特征 → 按页码合并为一份 PDF（调用 pdf skill）
3. 合并后的文件命名去掉页码后缀：`民事判决书_张三诉李四民间借贷纠纷.pdf`

### 已有规范命名的文件

- 如源文件中已有合规命名的文件（如 `（2025）苏0981民初1234号民事判决书.pdf`），直接复制不重命名
- 在映射表中标注"保持原名"

### 同名文件处理

- 如目标目录已存在同名文件，追加 `_02`、`_03` 后缀
- 在映射表中标注"同名冲突已重命名"

### OCR 失败的图片

- 标记为 `待确认/OCR失败_原文件名`
- 不移除、不删除、不强分类
- 在报告中汇总列出，提示用户手动查看

## 执行约束

1. **只读原则**：绝不修改、删除、覆盖任何原始文件
2. **全量复制**：所有原始文件都会被复制到输出目录，不存在"漏掉"的文件
3. **格式保持**：复制后的文件格式不变（不转换 PDF/Word/图片格式，除非多页合并场景）
4. **编码安全**：文件名使用 UTF-8 编码，中文文件名不出乱码
5. **路径安全**：使用绝对路径操作，防止路径穿越

## 与后续 Skill 协作

本 skill 是执行工作台的第一环节。整理完成后：

- `.file-registry.json` → 供 `case-analyzer`（案件分析）读取
- `01_生效文书/` → 供 `judgment-parser`（判决解析）处理
- `02_执行文书/` → 供 `enforcement-tracker`（执行进度跟踪）处理
- `03_流水材料/` → 供 `bank-flow-analyzer`（流水分析）处理
- `04_申请文书/` → 供 `application-generator`（申请书生成）参考
- `05_主体信息/` → 供 `enterprise-investigator`（企业调查）使用
